## 二、核心痛点

### 痛点 1: 成本结构不匹配

```
收入侧: 按条收费 (固定 ¥0.10-0.30/条，不限token数)
成本侧: 按 token 付费给上游中转站

一条 Sonnet 对话 3万token:  成本 ¥0.027 → 赚 ¥0.173
一条 Sonnet 对话 30万token: 成本 ¥0.27 → 亏 ¥0.07
一条 Gemini Pro 对话:      成本 ¥0.105 → 你卖 ¥0.10 → 直接亏

长对话不可控 → 利润不可控 → 可能单条亏几倍
```

### 痛点 2: 上游中转站不可控

- 质量参差不齐
- 频繁 429
- 模型掉包 (用便宜模型替换)
- 注水 (偷偷注入隐藏 token 增加计费)
- 价格波动
- 随时可能跑路

### 痛点 3: 无法锁死成本

- 用户对话长度 → 不可控
- 上游价格 → 不可控
- 上游是否注水 → 不可控
- 赚不赚钱 → 看运气

## 四、廉价中转站的真相: Antigravity OAuth

### 所有廉价 Claude/Gemini 都来自同一路径

```
Google 账号 (OAuth)
    │
    ▼
Antigravity (Google 的 AI 编程助手, 与 Anthropic 合作)
    │

    ├─→ Claude Sonnet 4.6
    ├─→ Claude Haiku 4.5
    ├─→ Gemini 3.1 Pro
    └─→ Gemini 3.1 Flash
```

**Sub2API 代码证实:**
- `antigravity_gateway_service.go` — 完整实现了 Antigravity 协议
- `gemini_quota.go` — 内置各等级账号配额
- `mapAntigravityModel()` — 模型映射: claude-sonnet-4-6, claude-opus-4-6, gemini-3.1-pro 等

### Antigravity 的计费方式

```
不是按 token 计费
只按 RPD (Requests Per Day) 限流

Google AI Ultra: 2000 RPD + 120 RPM
Google AI Pro:   1500 RPD + 120 RPM  
Google One Free: 1000 RPD + 60 RPM
AI Studio Free:  1500 RPD (Flash) + 50 RPD (Pro)

每次对话不管多少token → 只算 1 次 RPD
所以成本 = 固定月费 ÷ 月总请求数 = ¥0.005-0.012/条
```

```

按条收费 + 自建号池    = 固定成本 (不管对话多长)

号池成本: $100 (Ultra月费) ÷ (2000×30) = ¥0.012/条
你卖: ¥0.20-0.30/条
毛利: 94-96%
```

### 工作流

```
注册阶段 (PushOmni 虚拟手机):
1. 虚拟手机配置 SOCKS5 → SIM-IP
2. RPA 自动注册 Google 账号
3. 分散 14 天注册 (每天每 IP ≤1 个号)
4. 登录 AI Studio / Antigravity
5. 提取 OAuth Token
6. Token 导入 Sub2API

养号 (14天):
1. 每天 Google Search
2. 每天 YouTube
3. 每天 Gemini Web 对话
4. 逐步增加活动频率

运行:
1. Sub2API 请求 → 选可用号 → 经 SIM-IP 发请求
2. 429 → 自动冷却 → 切号
3. 死亡 → 自动标记 → 备用号顶替 → 触发补号
```

### 限额与风控

**Google AI Ultra 单号限制 (Sub2API 内置):**
- Shared RPD: 2000
- Shared RPM: 120
- 冷却时间: 5 分钟 (触发429后)

**每 IP 安全号数:**
- 1-3 个号: 完全安全
- 3-5 个号: 正常范围
- 5+ 个号: 风险递增

**50 SIM 的 IP 分配:**
- 每张 SIM = 1 个独立 IP
- 每 IP 只配 1 个活跃号 → 最安全级别

**Google 的限流信号类型 (Sub2API 解析):**
- RATE_LIMIT_EXCEEDED (短, <7s): 原地等待重试
- RATE_LIMIT_EXCEEDED (长, ≥7s): 标记模型限流 + 切号
- MODEL_CAPACITY_EXHAUSTED (503): 模型全局容量不足, 1s间隔重试
- Credits Exhausted: AI Credits 耗尽, 5小时冷却

1. **PushOmni 虚拟 Android 能否通过 Google Play Integrity?**
   - 这是号池能否跑通的关键
   - 需要实际测试

2. **Public Mobile 的 IP 被 Google 判定为什么类型?**
   - 加拿大运营商 (Telus 子公司) → 应是移动网络IP → 信任度应高于机房IP

3. **每 SIM IP 下注册 1 个号的长期存活率?**
   - 需要 30-60 天实际数据

4. **Antigravity Ultra 的实际 RPD 上限?**
   - Sub2API 写的是 2000，但实际可能有偏差
   - 需要压测确认

5. **加拿大 SIM 卡是否能过 Google 手机号验证?**
   - 作为注册时的验证手段
   - 或者需要额外接码平台
   - 注册环节最关键的一步

## 九、核心认知

1. **按条收费 + 中转站采购 = 不可持续** → 必须自建号池
2. **所有廉价中转站的 Claude 都来自 Antigravity OAuth** → Sub2API 已完整支持
3. **住宅/移动 IP 是核心资产** → 不是代码、不是软件
4. **Google 号是耗材不是资产** → 流水线化注册 + 自动补位
5. **加拿大身份是独特优势** → 合法宽带 + 合法手机套餐
6. **PushOmni + SIM 卡 = 号池工厂** → 50 IP + 50 手机 = 日产 10万条
7. **号池固定成本 → 利润可预测** → 跟对话长度无关
